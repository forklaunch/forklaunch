## Overview

ForkLaunch is a developer tool for building applications with idiomatic building blocks and no abstraction layers. Create composable primitives that fit your stack with no magic and no lock-in.

## AI Builds. ForkLaunch Protects.
### AI-Assisted Development

#### Let AI Write Code, Let the Compiler Catch Mistakes

ForkLaunch enables AI-first development by enforcing architecture through the compiler, not runtime abstractions.

#### AI Implements While ForkLaunch Enforces

- **Use any AI assistant**: Claude Code, Cursor, or any AI tool to generate services, endpoints, and business logic
- **AI writes implementation**: Let AI handle the code generation while ForkLaunch enforces architectural constraints
- **Compiler catches mistakes**: TypeScript shows every call that breaks after a change
- **Integration bugs caught early**: Problems are caught at compile-time, before deployment

#### MCP Context for AI Tools

- **Autogenerated MCP servers**: Your schemas automatically generate MCP servers
- **Perfect context**: AI tools get necessary context about your APIs automatically
- **System understanding**: AI understands your system architecture without manual documentation

#### Catching Errors AI Overlooks

- **Type-safe contracts**: Every API contract is enforced at compile-time
- **Breaking changes visible**: Change an API and TypeScript shows every caller that breaks
- **Compile-time validation**: Integration bugs are caught before they reach production

## Confidently Make Changes
### Safe Refactoring

#### Local Dev Matches Production

Run `forklaunch dev` and get a full production-like environment in seconds:

- **No manual setup**: No need to set up databases manually or configure environment variables
- **Docker Compose generated**: ForkLaunch manages all infrastructure automatically
- **Same constraints**: Your local environment carries the same constraints as production
- **Missing database?**: The compiler tells you before you deploy

#### Infrastructure as Code

Define databases, caches, and queues in TypeScript and ForkLaunch handles implementation:

- **TypeScript definitions**: Define infrastructure in code, not YAML
- **Automatic generation**: Manifest generates docker-compose for local dev and Pulumi for production
- **Type-safe resources**: All infrastructure resources are type-checked

#### Isolated Environments

- **No port conflicts**: Services run with isolated databases and caches
- **No shared state**: Each service has its own dedicated resources
- **Spin up multiple projects**: Run multiple projects simultaneously without interference

### Refactor Without Fear
Make changes confidently; production stays stable.

#### Break Builds at Compile-Time, Never Production

- **Compiler-time checks**: Catch broken architecture, infrastructure, dependencies, and integrations before deploying
- **Automatic validation**: ForkLaunch validates your changes before they reach production
- **Evolve confidently**: Change your system without worrying about breaking production

#### Architecture Evolution

Convert services to workers, extract shared libraries, split monoliths: ForkLaunch catches problems ahead of deployment:

- **Services to workers**: Convert any service to a background worker while keeping the same code structure and switching to a queue-based execution model
- **Extract libraries**: Extract common logic to shared libraries that you can import from any service, and TypeScript resolves the types
- **Split monoliths**: Break apart large services and the compiler ensures all dependencies are updated

#### Live-Typed SDKs

Services import each other like libraries with full type safety:

- **Automatic updates**: Live-typed SDKs update automatically when you change APIs
- **Editor autocomplete**: Import a live SDK and your editor autocompletes methods, parameters, and response types
- **Break builds, not production**: Refactoring breaks builds during development, never in production

### Hot Reload and Instant Updates

Change code and everything updates instantly:

- **File watching**: Automatic file watching configured out of the box
- **Hot module replacement**: See changes without full rebuilds
- **Restart individually**: Restart services individually without rebuilding everything

### Sensible Defaults, Not Enforced Structure

- **Clean patterns**: Generated services follow clear separation of concerns
- **Team consistency**: Optional but helpful for maintaining consistency
- **Not mandatory**: Structure is suggested, not enforced
- **Bring your own**: Integrate with existing codebase patterns

## Architecture as Code
### Define your system once. Everything else stays in sync.

#### Change the Schema, The Entire System Updates

Define your API contracts once using TypeScript validator schemas, and everything else updates automatically.

#### What You Get From One Schema

Write idiomatic validator schemas and get:

1. **TypeScript types**: Compile-time safety for all data structures
2. **Runtime validation**: Automatic request validation
3. **OpenAPI specs**: API documentation generated automatically
4. **Typed client SDKs**: Fully typed client libraries for consuming your APIs

#### API Contracts Enforced at Compile-Time

- **TypeScript validator schemas**: Every request, response, and event type is an enforced contract
- **Endpoint changes break builds**: Change an endpoint, and every service calling it shows compile errors
- **No runtime surprises**: Problems surface at compile time, not in production

#### Services Import Each Other Like Libraries

- **Full type safety**: Services import each other with complete TypeScript type checking
- **Library-like imports**: No special SDK generation; just TypeScript module imports
- **Editor support**: Full autocomplete, type hints, and error checking in your IDE

#### Real-Time Type Safety

WebSocket and SSE endpoints get the same type safety as REST:

- **Typed WebSocket messages**: Full type safety for real-time protocols
- **AsyncAPI specs**: Automatically generated AsyncAPI documentation
- **Event validation**: Real-time events are validated just like HTTP requests

### Breaks Surface at Compile Time

When an endpoint or event changes:

- Every dependent service fails at compile time, not in production
- TypeScript shows exactly which calls need updating
- No hidden integration failures
- Safe refactoring across your entire system

## Grow Without Rewriting
### Scale Up Without Lock-In

#### Works With Any TypeScript Framework

- **Bring your own framework**: Use ForkLaunch's Express framework with idiomatic validators, or integrate with your existing setup
- **Live-typed SDKs work anywhere**: SDKs work with any TypeScript framework
- **No framework lock-in**: Switch frameworks without changing your SDKs

### Full CloudWatch Access

- **Standard AWS services**: Full access to CloudWatch and standard AWS services
- **No proprietary abstractions**: Nothing hiding what's actually running
- **Complete visibility**: See everything in your AWS console

### Export to Pulumi Anytime

One command exports your infrastructure as Pulumi TypeScript:

- **Standard Infrastructure as Code**: No proprietary formats
- **Pulumi TypeScript**: Industry-standard IaC tool
- **Take the code**: Export, extend, or walk away at any time
- **No hidden dependencies**: Everything is standard Pulumi

### Deployment Options

Deploy to your AWS account with full control, or use managed cloud:

- **Your AWS**: Deploy to your own AWS account with complete control
- **Managed cloud**: Use ForkLaunch's managed infrastructure
- **Standard services**: ECS, RDS, ElastiCache, ALB, SQS; all standard AWS
- **Switch anytime**: Move between deployment options without code changes

### Compliance Templates

SOC2, HIPAA, and PCI templates automatically configure:

- **Audit logging**: Compliance-ready logging configured automatically
- **Encryption**: Data encryption at rest and in transit
- **IAM policies**: Security policies configured to compliance standards
- **Start compliant**: Meet regulatory requirements from day one

### Live-Typed SDKs Are Removable

- **Just TypeScript imports**: SDKs are standard TypeScript, not proprietary runtime dependencies
- **Remove anytime**: Take them out and your services keep running
- **Compiler enforces contracts**: Type safety comes from TypeScript, not a runtime framework
- **No runtime lock-in**: The compiler enforces contracts, not a proprietary runtime
